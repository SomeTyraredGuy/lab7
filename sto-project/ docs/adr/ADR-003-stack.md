# ADR-003: Вибір технологічного стеку Backend (Node.js + NestJS)

## Статус: Accepted

## Контекст:
Потрібно розробити Backend для MES-системи, що надаватиме REST API для веб-порталу менеджера та планшетів операторів. Операторам потрібне оновлення інформації на екранах у реальному часі (якщо план змінився, екран має оновитись без перезавантаження).

## Рішення:
Використовувати Node.js з фреймворком NestJS та мовою TypeScript для розробки Backend.

## Обґрунтування:
1.	Асинхронна (Event-driven) природа Node.js ідеально підходить для реалізації WebSockets, що необхідно для оновлення статусів у цеху в реальному часі.
2.	NestJS надає строгу архітектуру "з коробки", що стандартизує код і спрощує підтримку.
3.	TypeScript виявляє більшість помилок типізації ще до запуску коду, що критично для виробничої системи.

## Альтернативи:
| Варіант | Чому відхилено |
| :--- | :--- |
| Python + Django | Синхронна природа гірше справляється з великою кількістю постійних WebSocket-з'єднань від терміналів операторів. |
| Java + Spring Boot | Довший час написання коду (boilerplate), споживає значно більше оперативної пам'яті на сервері порівняно з Node.js. |

## Наслідки:
### Позитивні: Можливість використовувати одну мову (TypeScript) як на Frontend (React), так і на Backend.
### Негативні: Node.js є однопоточним, тому важкі обчислення (наприклад, генерація місячного Excel-звіту) доведеться виносити в окремі Worker-процеси.
