# ADR-001: Вибір СУБД (PostgreSQL для MES-системи)

## Статус: Accepted

## Контекст:
Система повинна обробляти близько 1000 виробничих завдань на добу. Одночасно в системі працюють близько 50 співробітників (менеджери, оператори). Дані (залишки сировини, статуси партій, результати перевірок якості) є строго структурованими та взаємопов'язаними. Потрібна сувора ACID-транзакційність, щоб уникнути ситуації, коли сировина списана зі складу, але партія не створена в цеху. Бюджет на ліцензії СУБД відсутній.

## Рішення:
Використовувати PostgreSQL 16 як основну реляційну базу даних.

## Обґрунтування:
1.	Повна підтримка ACID-транзакцій гарантує консистентність даних між складом та виробництвом.
2.	Вбудована підтримка типу JSONB дозволить гнучко зберігати специфічні параметри налаштувань для різних типів верстатів без зміни схеми БД.\
3.	Безкоштовна ліцензія (Open Source) задовольняє бюджетні обмеження.

## Альтернативи:
| Варіант | Чому відхилено |
| :--- | :--- |
| MongoDB (NoSQL) | Не підходить для складних реляційних зв'язків (Замовлення -> План -> Сировина -> Сертифікат якості). Немає суворої гарантії транзакцій для фінансово важливих складських даних. |
| Microsoft SQL Server | Висока вартість ліцензії для комерційного використання. |

## Наслідки:
### Позитивні: Надійне збереження даних, можливість легко інтегруватися з BI-системами для побудови звітів.
### Негативні: Потребує налаштування резервного копіювання (бекапів) та оптимізації індексів при збільшенні обсягу архівних замовлень.
